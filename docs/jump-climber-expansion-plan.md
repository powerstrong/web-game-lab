# Jump Climber Expansion Plan

`jump-climber`는 현재 사용자에게 노출되는 이름 기준으로
`말랑프렌즈 점프`다.

이 문서는 이 게임을 단일 캐릭터 솔로 점프 프로토타입에서
대표 타이틀 수준의 확장 가능한 게임으로 키우기 위한 작업 계획이자
현재 구현 상태를 남기는 작업 메모다.

## Current Product Name

- 앱 / 플랫폼 이름: `말랑프렌즈 아케이드`
- 게임 이름: `말랑프렌즈 점프`

## Goal

핵심 목표는 아래 경험을 하나의 일관된 흐름으로 묶는 것이다.

1. 플레이 전에 3개의 캐릭터 중 하나를 고를 수 있어야 한다.
2. 원하면 사용자 얼굴 사진을 캐릭터 얼굴 영역에 합성해 프리뷰할 수 있어야 한다.
3. 2인 플레이가 가능해야 한다.
4. 모바일에서도 짧은 설정 후 바로 플레이에 들어갈 수 있어야 한다.

## Implemented So Far

### 1. Character Selection

현재 기본 캐릭터 3종이 구현되어 있다.

- `모찌 토끼`
- `푸딩 햄스터`
- `말랑 병아리`

구현 상태:

- 설정 화면에서 3개 캐릭터 선택 가능
- 대표 썸네일은 오른쪽을 보는 스프라이트 사용
- 상태별 스프라이트 연결 완료
  - `jump_neutral`
  - `jump_left`
  - `jump_right`
  - `fall_neutral`

### 2. Face Swap Customization

현재 브라우저 로컬 기반 얼굴 합성 프리뷰가 구현되어 있다.

구현 상태:

- 얼굴 사진 업로드
- 얼굴 on/off 토글
- 얼굴 위치 조정
- 얼굴 영역 크기 조정
- 얼굴 안 사진 확대 조정
- 캐릭터 본체 크기 조정

현재 컨트롤 의미:

- `얼굴 확대`: 얼굴 원형 영역 자체 크기
- `사진 확대`: 원형 영역 안의 사진 확대
- `캐릭터 크기`: 캐릭터 본체 전체 확대

현재 한계:

- 로컬 싱글 플레이에서는 완전 동작
- 방 플레이에서는 내 기기에서의 프리뷰/반영 중심
- 얼굴 이미지를 상대 기기와 완전히 공유하는 구조는 아직 없음

### 3. Gameplay

구현 상태:

- 싱글 플레이 가능
- 로컬 2P 플레이 가능
- 방 플레이에서는 2개 기기가 같은 맵에 들어가는 shared-map 멀티플레이 프로토타입 가능
- 카메라는 낮은 플레이어 기준 추적
- 플레이어별 최고 높이 표시
- 한 명 탈락 후 다른 한 명 계속 진행
- 시작 위치를 바닥보다 약간 위로 보정

### 4. Mobile UX

구현 상태:

- 설정 화면 / 실제 게임 화면 분리 완료
- 플레이 중 설정 UI가 화면을 차지하지 않음
- 모바일에서 전체 화면 좌/우 반분 터치 입력 유지
- 맵 폭이 모바일 화면 안에 들어오도록 자동 축소

### 5. Art Integration

현재 연결된 자산:

- 캐릭터 12장
- 하늘 배경
- 플랫폼 4종
- 부스트 2종
- 움직이는 장식 기물 2종

구현 상태:

- 배경 적용 완료
- 플랫폼 PNG 렌더 적용 완료
- 부스트 PNG 렌더 적용 완료
- 이동 / 회전 플랫폼에 장식 기물 적용 완료

## Remaining Issues

### 1. Multiplayer Lag / Flicker

현재 가장 큰 품질 이슈다.

지금까지 한 것:

- 서버 authoritative 구조 유지
- 입력 즉시 전송 비중 확대
- 서버 틱 단축
- DOM 재사용 시작
- 원격 엔티티 보간 추가
- 로컬 플레이어 수평 예측 추가

아직 남은 문제:

- 체감상 여전히 멀티플레이 지연이 느껴짐
- 특정 순간 스프라이트 갱신이 어색하게 보일 수 있음
- 원격 플레이어와 플랫폼 스냅샷 보간을 더 안정화할 필요가 있음

다음 단계:

- 로컬 플레이어 예측 강화
- 원격 엔티티 스냅샷 버퍼 보간
- 불필요한 `innerHTML` 갱신 제거

### 2. Effects / Audio

아직 점프 손맛이 충분히 살아나지 않았다.

다음 단계:

- 점프 이펙트
- 착지 이펙트
- 부스트 획득 이펙트
- 기본 효과음

### 3. Face Alignment

현재 얼굴 박스는 캐릭터별 대략값으로 맞춰져 있다.

다음 단계:

- 토끼 / 햄스터 / 병아리 각각에 맞는 미세 조정
- 큰 확대값에서도 어색하지 않게 보정

### 4. Deployment Structure

현재 정적 사이트와 방 API 워커가 분리되어 있다.

현재 상태:

- 프론트: `web-game-lab.powerstrong.workers.dev`
- 방 API: `game-lobby.powerstrong.workers.dev`

다음 단계:

- 이 구조를 유지할지
- 메인 origin으로 통합할지
- Cloudflare 운영 체크리스트를 문서화할지
정리 필요

## Current Architecture Notes

### Solo / Local Play

- 브라우저에서 `prototypes/jump-climber/index.html` 직접 열면
  싱글 플레이 또는 로컬 2P 테스트 가능

### Room Play

- 각 기기가 캐릭터 1명씩 맡는다
- 같은 맵 상태를 공유한다
- 서버가 authoritative하게 상태를 계산한다
- 클라이언트는 입력 송신 + 스냅샷 렌더를 담당한다

이 구조는 프로토타입으로는 충분하지만,
완성도를 더 올리려면 예측/보간을 더 다듬어야 한다.
