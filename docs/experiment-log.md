# Experiment Log

짧게 만들고 바로 판단하기 위한 로그입니다.

## 말랑프렌즈 점프

### Hypothesis

귀여운 캐릭터 선택과 얼굴 커스터마이즈, 그리고 같은 맵을 공유하는 2기기 점프 플레이를 결합하면
짧은 시간 안에도 강한 애착과 반복 플레이 욕구가 생길 수 있다.

### What worked

- 캐릭터 스프라이트를 실제 자산으로 교체한 뒤 게임 톤이 빠르게 살아났다
- 설정 화면과 플레이 화면을 분리하니 모바일에서 훨씬 다루기 쉬워졌다
- 같은 맵 shared-map 멀티플레이는 방향성이 분명하고 재미 포인트가 있다
- 움직이는 / 회전하는 플랫폼이 들어가니 맵이 덜 평평하게 느껴진다

### What felt weak

- 멀티플레이는 아직 렉과 스냅샷 깜빡임이 체감된다
- 얼굴 합성 위치는 캐릭터별로 더 정교한 보정이 필요하다
- 점프 손맛용 이펙트와 소리가 없어 약간 밋밋하다

### Keep / Kill / Mutate

- `Keep`: 말랑프렌즈 아케이드 브랜드, 말랑프렌즈 점프, 설정/플레이 화면 분리, shared-map 멀티플레이
- `Keep`: 실제 캐릭터/배경/플랫폼 아트 연결
- `Mutate`: 멀티 렌더 구조는 DOM 재사용 + 예측 + 보간 중심으로 더 다듬기
- `Mutate`: 얼굴 커스터마이즈는 로컬 프리뷰 중심에서 향후 공유 가능한 구조까지 검토

### Next version constraint

다음 버전에서는 네트워크 품질을 개선하되,
완전한 rollback/resimulation처럼 프로토타입 규모를 크게 넘는 복잡도는 아직 넣지 않는다.
